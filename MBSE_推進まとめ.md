## 2. 確立したルール

### 2-1. レイアウトルール

| ルール | 内容 |
|---|---|
| 縦方向レイアウト | 上位要素（親）が上段・下位要素（子）が下段 |
| 親の幅 | 子要素群をカバーする幅に設定 |
| 矢印方向 | 上から下へ垂直に流れる構成 |

> **理由**: 左右配置では矢印がごちゃつくため、縦方向にすることで親子関係が一目でわかる。

### 2-2. 命名ルール

| 種別 | ルール | 例 |
|---|---|---|
| ステークホルダ要求 | `SR-XXX: 動詞＋目的語` | SR-001: 指定速度での搬送操作 |
| システム要求 | `SysR-XXX: 対象＋数値` | SysR-001: 速度制御精度 ±2% |
| ブロック | 日本語・役割名 | 速度コントローラ、安全監視モニタ |
| ポート属性 | `in_` / `out_` プレフィックス | in_目標速度、out_PWM指令 |

### 2-3. トレーサビリティルール

| リンク種別 | 用途 | 方向 |
|---|---|---|
| `deriveReqt` | SR → SysR（導出関係） | SysR → SR |
| `satisfy` | ブロック → SysR（実現関係） | ブロック → SysR |
| `InformationFlow` | ブロック間の信号の流れ（IBD） | 送信元 → 受信先 |
| タグ付き値 | 検証結果の記録 | Status / VerifiedBy / VerifiedResult |

## 4. 図の種類と目的

| 図 | 目的 |
|---|---|
| 要求トレーサビリティ図 | SR → SysR の deriveReqt 関係を縦方向に可視化 |
| BDD（ブロック定義図） | ブロック構造・属性（値プロパティ）・ポート（フロープロパティ）を定義 |

| 検証結果サマリ図 | 検証済み・未検証の一覧を EA で管理 |

## 6. Simulinkモデル構成

```
ConveyorSpeedControlSystem.slx
├── 操作パネル         … 速度設定値・停止信号・ステータス出力
├── 速度コントローラ   … 偏差計算 → PID → Saturation → Switch
├── 安全監視モニタ     … 電流比較（>しきい値）OR 緊急停止 → 停止指令
├── モータドライバ     … PWM × Gain(24V) → Switch → 電圧出力
├── DCモータ           … 速度変換 → 1次遅れ伝達関数（τ=0.5s）
├── 速度センサ         … 単位変換 + センサノイズ（σ=0.01）
└── 電流センサ         … 電圧 × Gain(0.4) → 電流値
```

**フィードバックループ**:
```
操作パネル → 速度コントローラ → モータドライバ → DCモータ
                ↑                                    ↓
            速度センサ ←────────────────────────────┘
                                    ↓
                          電流センサ → 安全監視モニタ → 停止指令
```

## 11. 教訓・注意点

| 事項 | 内容 |
|---|---|
| EAの線の削除 | 図から削除するとモデルからも消える場合あり。Ctrl+Zで即戻す |
| EAの配置変更 | `place_element_on_diagram` は既存要素の移動不可。新規図を作成して再配置 |
| コード管理 | MATLABコードはセッション内実行だけでなく `.m` ファイルとして保存する |
| 代数ループ | Simulink で代数ループが発生した場合は Memory ブロックで1ステップ遅延を挿入 |
| 言語統一 | EA・Simulink ともに日本語で統一するとトレーサビリティが追いやすい |
